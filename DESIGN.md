# DESIGN.md

funailog.com のデザイン方針。Phase をまたいで参照する上位の意思決定を記録する。

## ターゲット読者

ガジェット好き全般（技術背景不問）。プログラミング・旅行・ライフスタイル記事も扱うが、サイト全体のトーンはガジェット・デスクツール・スマートホームを軸に設計する。

## デザイン方向性: Adaptive Duo-tone

ライト/ダーク両モードで同等に美しいデザイン。どちらか一方を「主」としない。

- ライトモード: 柔らかいシャドウ、ライトグレー背景、温かみのあるニュートラル
- ダークモード: 微細なボーダー、深いグレー背景、コントラスト制御

## フォント

- 見出し: LINE Seed JP（ジオメトリック × 角丸の温かみ）
- 本文: システムフォントスタック
- コード: JetBrains Mono

## デザイン原則

1. **広告感を消す** — アフィリエイトは記事の文脈に溶け込むデザイン。PR バッジは維持するが、カード全体は記事トーンに統一
2. **両モードで美しく** — コンポーネントはセマンティックカラートークンで記述し、ライト/ダーク両方で意図した体験を提供
3. **両デバイスで一級** — モバイル・デスクトップの両方の体験を一切損なわない。設計はモバイルから始めてデスクトップに拡張するが、デザイン優先度は同等
4. **jp-ui-contracts 準拠** — 日本語タイポグラフィの行長制御、text-wrap、BudouX 統合
5. **コンポーネント駆動** — /styleguide ページで全パターンを検証（dev 環境のみ）。2 モード × 2 幅（モバイル/デスクトップ）で目視確認
6. **WCAG AA 準拠** — eslint-plugin-jsx-a11y（静的解析）+ pa11y-ci + axe-core（ビルド後 HTML 検証）の 2 層チェック

## 技術スタック

Astro v6 + React 19 + Tailwind CSS v4 + MDX。フレームワークは変えず、デザインシステム層をゼロから再構築する。
